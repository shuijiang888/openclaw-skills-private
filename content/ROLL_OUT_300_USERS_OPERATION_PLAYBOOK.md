# 300 人跨城市上线作战手册（v1.0）

**目标规模**: 约 300 人，分布于数十个城市，含直营与代理商  
**策略**: 分波次灰度 + Gate 准入 + 自动化运营

---

## 1. 上线总体策略

采用四波次发布，不一次性全开（与 `AGENT_OWNER_PRD_V3.md` 第 10 节一致）:

- Wave 1（种子试点，约 30 人，10%）
- Wave 2（跨城市扩面，约 90 人，30%）
- Wave 3（管理体系上线，约 210 人，70%）
- Wave 4（全量开放，约 300 人，100%）

每波次必须过 Gate，再进入下一波。

---

## 2. 波次目标与通过标准

## 2.1 Wave 1（种子组）
目标:
- 验证“输入 -> 动作卡 -> 回填”链路稳定
- 验证任务悬赏最小闭环

通过标准:
- 动作卡触达率 >= 95%
- 48 小时动作执行率 >= 35%
- 回填率 >= 55%
- 高优先级信号处理 SLA 达标率 >= 85%

## 2.2 Wave 2（区域扩面）
目标:
- 验证跨城市协作与差异化运营

通过标准:
- 关键链路错误率 <= 1.5%
- 48 小时动作执行率 >= 35%（不低于 Wave 1）
- P0 信号从入库到首轮处置 <= 120 分钟，达标率 >= 90%
- 越权访问告警为 0
- 用户投诉可在 24 小时内闭环

## 2.3 Wave 3（角色扩展）
目标:
- 主管、运营、管理员流程稳定
- 审核和策略调整机制跑顺

通过标准:
- 审核积压 <= 24 小时待审量阈值（建议 <= 50 条）
- 任务结算及时率 >= 95%
- 月度对账差异率 <= 1%
- RBAC 抽检通过率 >= 98%

## 2.4 Wave 4（全量开放）
目标:
- 进入稳定运营，开始经营优化

通过标准:
- 以下关键指标连续两周稳定:
  - 48 小时动作执行率 >= 40%
  - 回填率 >= 60%
  - 关键链路错误率 <= 1.0%
  - P0 信号首轮处置达标率 >= 92%
- 反作弊机制命中有效
- 异常恢复流程验证通过

---

## 2.5 指标口径（防歧义）

- 动作卡触达率: 成功送达用户的动作卡数量 / 计划送达数量
- 48 小时动作执行率: 48 小时内标记“已执行”的动作卡 / 已送达动作卡
- 回填率: 有执行结果回填的动作卡 / 已送达动作卡
- 关键链路错误率: 关键 API 请求失败数 / 关键 API 请求总数
- P0 信号处置时长: P0 信号入库到首次人工或系统处置完成的时间差

---

## 3. 上线前检查（Go/No-Go）

必须项:
1. 多租户隔离已验证
2. RBAC 权限矩阵生效
3. 备份与恢复演练通过
4. 微信/企微推送稳定
5. 审计日志完整可查
6. 故障降级策略可执行

任一未通过，不得扩容。

---

## 4. 日常运营机制（单人可执行）

## 4.1 每日动作
- 看“系统健康日报”（服务、队列、容量）
- 看“运营异常日报”（执行率低、噪音高、任务积压）
- 发布/调整当日重点任务

## 4.2 每周动作
- 复盘三类榜单: 质量榜、协作榜、进步榜
- 调整积分权重与任务模板
- 输出周报给关键管理角色

## 4.3 每月动作
- 成本复盘（AI 调用、存储、带宽）
- 线下核销与线上账本对账
- 权限审计（离职、调岗、越权）

---

## 5. 任务悬赏运营 SOP

1. 发布任务（目标、范围、奖励、截止）  
2. 自动触达目标人群  
3. 提交与 AI 初筛  
4. 人工验收与分级采纳  
5. 奖励发放与公示  
6. 成果归档进知识库

SLA 建议:
- 紧急任务: 4 小时首轮验收
- 常规任务: 24 小时内完成验收

---

## 6. 荣誉与激励策略（健康 PK）

### 6.1 榜单结构
- 质量榜: 被采纳率、价值分
- 协作榜: 助攻他人完成任务
- 进步榜: 环比提升幅度

### 6.2 荣誉播报
- 日播报: 轻量提醒
- 周播报: 战区亮点
- 月播报: 高光人物与团队

### 6.3 频控策略
- 同人同周重大荣誉推送 <= 1 次
- 群播报每日上限控制，避免打扰

---

## 7. 风险与应急

### 7.1 常见风险
- 用户输入质量下降
- 审核积压导致时效下降
- 刷分与互赞行为出现
- 多模态处理排队过长

### 7.2 应急动作
- 提高筛选阈值，先保质量
- 临时缩减推送频次，稳定体验
- 启用手动审核兜底队列
- 暂停高成本荣誉生成任务
