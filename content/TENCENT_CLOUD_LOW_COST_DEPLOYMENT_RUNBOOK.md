# 腾讯云低成本部署 Runbook（单人可运维）

**目标环境**: 中国内地，约 300 人试运行  
**部署原则**: 低成本、低复杂度、可扩展、可回滚

---

## 1. 最小可行部署拓扑

### 1.1 首发（单机）
- CVM 1 台（4C8G，100G SSD）
- Docker Compose 运行以下服务:
  - `api`（FastAPI / Next API）
  - `worker`（异步任务）
  - `postgres`（主库，建议启用 pgvector）
  - `redis`（缓存 + 队列）
  - `nginx`（网关 + HTTPS）
- COS 1 个存储桶（图片/音视频/文档）

### 1.2 扩展（第二阶段）
- 新增 1 台 worker 主机（只拆异步处理）
- 数据库仍单实例，先做备份和只读观察

---

## 2. 云资源采购清单

1. CVM（华南或华东）  
2. 云硬盘（SSD）  
3. COS（标准存储）  
4. 域名 + ICP 备案  
5. SSL 证书  
6. 云监控与告警（短信/微信）

---

## 3. 网络与安全基线

### 3.1 安全组
- 22: 仅固定管理 IP 可访问
- 80/443: 对公网开放
- 5432/6379: 不对公网开放，仅内网或本机

### 3.2 主机安全
- 禁止 root 密码登录
- SSH 密钥登录
- 定期升级系统补丁

### 3.3 应用安全
- JWT + 角色权限 + 数据分级
- 审计日志记录导出、权限变更、敏感访问
- 备份文件加密保存

---

## 4. 部署步骤（顺序执行）

1. 初始化主机（时区、NTP、Docker、Compose）
2. 上传代码并配置 `.env`
3. 执行数据库迁移与初始数据
4. `docker compose up -d`
5. 配置 Nginx 与 HTTPS
6. 配置微信/企微回调并启用签名校验、重放保护
7. 执行健康检查与链路验证
8. 执行“备案与合规上线清单”核验（见第 10 章）

---

## 5. 环境变量最小清单

- `APP_ENV`
- `DATABASE_URL`
- `REDIS_URL`
- `JWT_SECRET`
- `WECHAT_APP_ID`
- `WECHAT_APP_SECRET`
- `COS_BUCKET`
- `COS_REGION`
- `COS_SECRET_ID`
- `COS_SECRET_KEY`
- `AI_PROVIDER`
- `AI_BUDGET_DAILY_LIMIT`

如启用 CRM Integrated 模式，增加:
- `CRM_MODE=standalone|integrated`
- `CRM_BASE_URL`
- `CRM_CLIENT_ID`
- `CRM_CLIENT_SECRET`
- `CRM_WEBHOOK_SECRET`
- `CRM_SYNC_CRON`
- `CRM_WRITEBACK_ENABLED`

说明:
- 默认 `CRM_MODE=standalone`，不依赖外部 CRM。
- `CRM_WRITEBACK_ENABLED` 默认关闭，灰度验证后开启。

## 6. 运维自动化（必须）

### 6.1 每天
- 数据库自动备份（本地 + 对象存储）
- 日志轮转
- 关键链路巡检（登录、上报、推送、回填）
- 告警汇总推送到企业微信

### 6.2 每周
- 备份恢复演练
- 成本报表（模型调用、存储、带宽）
- 异常账号与刷分报告
- 密钥轮换检查（JWT、微信/企微、COS、CRM）

---

## 7. 成本控制策略

1. 高价值任务才走重模型  
2. 普通信号走规则或轻模型  
3. 音乐/视频荣誉生成分级触发  
4. 图片和视频做生命周期管理（冷存、归档）  
5. 默认采用“专业权威”模板型荣誉内容，减少高成本生成任务比例  

---

## 8. 灾备与回滚

### 8.1 回滚策略
- 保留最近 3 个可用镜像 tag
- 数据变更脚本可逆
- 配置变更有版本记录

### 8.2 事故分级
- P0: 主链路不可用或越权泄露 -> 立即降级与封禁高风险能力
- P1: 局部功能异常 -> 重试与补偿队列
- P2: 体验优化类 -> 纳入下个迭代

## 10. 备案与合规上线清单（中国内地）

### 10.1 备案与域名
- [ ] ICP 备案完成并可查询
- [ ] 备案号在站点底部规范展示
- [ ] 已备案域名与实际解析地址一致

### 10.2 微信/企微配置
- [ ] 回调 URL、可信域名、业务域名已完成平台配置
- [ ] 回调签名校验与 nonce/timestamp 防重放生效
- [ ] 应用密钥仅保存在安全配置，不写入日志

### 10.3 PIPL 与数据处理
- [ ] 隐私政策与用户协议已上线并可访问
- [ ] 多模态内容处理用途、保留期限、删除策略已声明
- [ ] 对外模型调用字段已做最小化与脱敏策略

### 10.4 审计与应急
- [ ] 高敏访问与导出全量审计
- [ ] P0 故障联系人与应急流程可执行
- [ ] RPO/RTO 已定义并在演练中验证
