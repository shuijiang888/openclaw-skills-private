**文档版本**：1.0  
**适用范围**：企业选型、立项汇报、实施规划与产品路线图对齐  
**关联实现**：`profit-web` 演示系统（一期 MVP）

## 第一章 价值主张

本章回答：系统为什么值得上、对谁产生什么可感知价值。价值实现程度与**数据质量、流程纪律、组织配套**正相关（见第二章）。

### 1.1 对企业的整体价值

| 维度 | 价值说明 |
|------|----------|
| **盈利可视化** | 将「报价—成本—毛利—审批—成交」从表格与口头经验，转为可查询、可对比、可复盘的数据链路，降低利润黑箱。 |
| **决策一致性** | 通过系数规则、分层审批与分流策略，减少「同单不同价、同级不同批」的随意性，把隐性制度显性化。 |
| **响应与合规** | 标准场景压缩报价与初审周期；敏感折扣自动升级角色留痕，兼顾效率与内控。 |
| **组合经营** | 盈利罗盘与预警对策把单笔报价上升到产品线/客户结构视角，支撑资源投放与收缩决策。 |

**一句话**：在「卖得出去」和「赚得回来」之间搭一座**可度量、可审批、可复盘**的桥。

### 1.2 对老板 / 决策层的价值

- **看结构**：哪些客户、产品线在贡献真利润，哪些在消耗资源（罗盘象限 + 集中度/账期等预警）。  
- **控风险**：低毛利、长账期、大客户依赖等触发对策提示，便于拍板资源与信用政策。  
- **要结果**：审批链与成交价留痕，便于追问「谁、在何依据下」定了价，而不是只看最终合同价。  
- **定节奏**：迭代路线可分期投入（见第三章），避免一次性大而无当的信息化项目。

### 1.3 对销售管理者（总监 / VP）的价值

- **管道透明**：待审批队列、折扣档位与所需角色一目了然，减少跨级扯皮。  
- **辅导有据**：系数与对标帮助复盘「为什么这单毛利薄」，用于培训与模板优化。  
- **战略单护航**：人机协同通道强制关键单走评审，避免自动化误伤大客户或复杂定制。  

### 1.4 对一线销售 / 报价员的价值

- **少算错**：成本拆解 + 连乘系数 + 建议价与毛利率试算，降低 Excel 手误。  
- **少跑腿**：满足规则时自动/快速通道，满足条件时系统提示下一审批人。  
- **好解释**：对标历史、竞品占位、胜率（规则版）提供「跟客户怎么说」的素材。  

### 1.5 对财务 / 运营管理的价值

- **口径对齐**：推动明确「报价毛利」与「核算毛利」的差异及映射关系（演示期为简化口径，生产期需与财务科目对齐）。  
- **主数据责任**：客户评级、账期、成本要素来源可追溯，为对账与审计提供锚点。  
- **预测辅助**：成交与丢单数据沉淀后，可支持滚动预测与预算（依赖后续迭代）。  

### 1.6 对信息化 / IT 的价值

- **边界清晰**：一期可独立演示；二期以 API/集成方式对接 ERP，而非推翻重来。  
- **可演进**：规则引擎、审批流、指标层可替换为更成熟组件，演示架构可迁移。

## 第二章 落地部署的业务、数据与管理依赖性及前置要求

本章用于**立项评估与实施排期**：缺哪类前置条件，就应把对应工作列入项目第一期或企业变革任务。

### 2.1 业务侧前置条件

| 要求 | 说明 | 不满足时的典型后果 |
|------|------|---------------------|
| **报价与成本口径共识** | 明确报价毛利是否含期间费、是否含税、是否与财务结转一致 | 系统数字与财报对不上，信任崩塌 |
| **审批授权制度** | 折扣/特价与组织职级对应关系书面化（可配置） | 系统流程与真实签字流两张皮 |
| **客户/产品分级标准** | 战略客户、标品定义、小额订单阈值等 | 自动分流误杀或漏控 |
| **主数据责任人** | 谁维护客户、物料、价格清单 | 数据陈旧，报价失真 |

### 2.2 数据与系统集成依赖

| 层级 | 依赖性 | 一期演示 | 生产建议 |
|------|--------|----------|----------|
| **成本数据** | BOM、工艺工时、采购价、制造费用分摊 | 手工录入/导入即可跑通 | 对接 ERP/MES 或定期同步 |
| **历史报价与成交** | 相似物料历史价、中标/丢单记录 | 种子数据 + 手工维护对标 | CRM/CPQ/合同系统回写 |
| **外部价格** | 大宗材料行情、竞品价格 | 占位字段 | 行情 API 或采购情报 |
| **组织与账号** | 汇报关系、角色、权限 | 演示用 Header 角色 | SSO、组织架构同步 |

### 2.3 管理与组织配套

- **变革管理**：销售需接受「还价即留痕、超权限必升级」，否则系统被绕过。  
- **考核对齐**：若 KPI 仍唯收入不唯毛利，一线会倾向压价绕过规则；需管理层表态。  
- **运维与迭代**：指定产品负责人 + IT 对接人，规则变更走变更记录（演示期可简化）。  

### 2.4 合规与安全（生产环境）

- 敏感字段（成本、底价）分级权限与审计日志。  
- 跨境/行业监管（如出口报价、关联定价）需法务合规评审。  
- 备份、容灾与数据库迁移策略（SQLite 仅适合演示；生产建议 PostgreSQL 等）。  

### 2.5 小结：最小可行前置包（MVP 上线）

1. 书面化的审批折扣档位 + 客户/产品分级定义。  
2. 至少一条业务线愿意**连续 4–8 周**用系统做单（可并行 Excel 双轨一周过渡）。  
3. 成本数据**可先导入后自动**，不必等 ERP 百分百完美。

## 第三章 后续迭代更强版本的前瞻计划与价值预期

按**价值递增与依赖递增**排序；阶段命名可与内部财年对齐。

### 3.1 近期（3–6 个月）：MVP+ 强化运营

| 方向 | 内容 | 价值预期 |
|------|------|----------|
| **集成** | 主数据从 ERP/Excel 定时同步；报价单回写订单草稿 | 减少重复录入 50%+，口径争议下降 |
| **规则可配** | 系数方案版本化、按客户群/产品线生效 | 业务可自助调参，IT 工单减少 |
| **报表** | 审批时效、自动通道占比、低毛利单列表导出 | 管理例会可直接用系统出数 |
| **权限** | 真登录、角色库、字段级脱敏 | 满足内控与审计基本要求 |

### 3.2 中期（6–18 个月）：预测与协同

| 方向 | 内容 | 价值预期 |
|------|------|----------|
| **数据驱动胜率** | 基于历史成交训练或校准加权模型 | 报价会胜率评估从「感觉」到「可追溯」 |
| **情景模拟** | 材料涨跌、汇率、批量阶梯对建议价的一键情景 | 采购与报价联席决策更快 |
| **工作流引擎** | 可配置多级会签、并行审批、代理 | 适配集团化组织 |
| **移动端** | 审批 App / 企业微信待办 | 决策时延从「天」到「小时」 |

### 3.3 远期（18 个月+）：智能与生态

| 方向 | 内容 | 价值预期 |
|------|------|----------|
| **策略推荐** | 结合库存、产能利用率的报价区间建议 | 从「单点最优」走向「全局更优」 |
| **与 PLM/CRM 深度联动** | 配置即报价（CPQ 化） | 复杂定制行业标配能力 |
| **行业模板库** | PCB、机加工、装备等垂直包 | 缩短上线周期，降低实施成本 |

### 3.4 价值预期汇总（对内沟通口径）

- **近期**：「能用、少错、可审计、少录入」。  
- **中期**：「更快决策、更准预测、更贴组织」。  
- **远期**：「系统参与经营策略，而不只是算价工具」。  

### 3.5 智能体、分析与「智慧大脑」（与产品路线并行）

面向 **Agent 化**、**可解释的分析与决策**、**老板与各级的分角色外脑** 三条改进主轴，已单独整理为 **`docs/AGENT_AND_INTELLIGENCE_ROADMAP.md`**（与当前 `profit-web` 助手/罗盘/审批实现逐条对齐，含分期 A1–A3、B1–B3 与角色体验表）。选型汇报可将本章作为价值叙事，将该文档作为技术演进附件。
