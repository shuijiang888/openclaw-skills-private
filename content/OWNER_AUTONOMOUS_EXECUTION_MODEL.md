# Owner 单人决策 + 多 Agent 协同执行模型（v1.0）

**适用场景**: 你是唯一 Owner，没有完整团队；需要快速从构想到上线运营。  
**目标**: 不增加组织负担，仍可达成端到端闭环交付。  
**术语说明**: 本文“主控 Agent / 子 Agent”指协作执行角色；与运行时软件架构中的 Agent Core 组件（见 `content/AGENT_STANDALONE_INTEGRATED_ARCHITECTURE.md`）不是同一概念。

---

## 1. 角色定义（极简但完整）

### 1.1 Owner（你）
只保留四类决策:
1. 业务优先级
2. 成本上限
3. 风险阈值
4. 扩容节奏

### 1.2 主控 Agent（总负责）
- 负责全局架构、任务拆分、质量验收、版本集成
- 对你输出“结论 + 风险 + 下一步”，不输出冗长过程

### 1.3 子 Agent（并行执行）
- 产品与文档 Agent
- 后端与数据 Agent
- 前端与交互 Agent
- 测试与运维 Agent
- 运营增长 Agent

---

## 2. 工作方式（默认并行）

### 2.1 迭代节奏
- 每轮迭代遵循: `规划 -> 实现 -> 验证 -> 发布 -> 复盘`
- 主控 Agent 汇总子 Agent 产物，给出唯一集成版本

### 2.2 交付物清单
每轮至少包含:
1. 可运行版本（或可验证原型）
2. 更新日志（变更点、影响面、回滚方法）
3. 风险清单（已知问题 + 缓解措施）
4. 下一轮建议（可直接决策）

---

## 3. 决策协议（减少沟通成本）

### 3.1 默认授权
你已授权主控 Agent 在以下范围自动决策:
- 技术实现路径
- 文档结构与规范
- 低风险体验调整
- 非破坏性重构

### 3.2 必须回报的决策
以下事项必须单独报告并建议你确认:
- 新增明显成本项（云资源、第三方 API）
- 涉及合规或数据边界变更
- 对外发布口径与品牌层决策
- 可能影响存量用户体验的大改动

---

## 4. 质量门禁（Gate）

### 4.1 开发 Gate
- 单功能可验证
- 日志可追踪
- 失败可回滚

### 4.2 发布 Gate
- 核心链路可用（输入 -> 动作 -> 回填）
- 关键告警可达
- 权限隔离有效

### 4.3 运营 Gate
- 日报自动生成
- 反作弊规则生效
- 任务与积分结算可审计

---

## 5. 成本控制协议（单人友好）

1. 开源优先，托管后置  
2. 单机可运行优先，集群后置  
3. 高耗 AI 能力按价值触发，不全量默认开启  
4. 所有自动化生成能力必须有预算上限和频控  

---

## 6. 异常处理机制（避免 Owner 被拖死）

### 6.1 P0 故障
- 主链路不可用、数据越权、核心服务宕机
- 主控 Agent 立即降级并输出简报: 影响范围、临时策略、恢复 ETA

### 6.2 P1 故障
- 某子功能异常、性能抖动、少量同步失败
- 自动重试 + 队列补偿 + 日内复盘

### 6.3 P2 事项
- 体验小缺陷、文案优化
- 统一进入下一轮迭代池

---

## 7. 你的一周最小动作

你每周只做三件事:
1. 看 1 份周报（指标 + 风险 + 决策建议）
2. 拍 3 个决策（优先级、预算、节奏）
3. 定 1 个下周主目标（例如执行率提升）
