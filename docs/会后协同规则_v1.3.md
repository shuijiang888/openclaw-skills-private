---

## 3. 强制执行规则

1. 每次 push 后必须在协同通道同步 SHA 与变更点。  
2. API 层权限是唯一准入，不得以“前端隐藏按钮”替代权限判定。  
3. 跨系统联动必须走 API 契约，禁止跨壳直引业务组件。  
4. 任何发布前必须通过 Nginx 白名单与 SHA 门禁。  

---

## 4. 发布门禁（新增强制）

### 4.1 Nginx 白名单门禁

发布前检查远端 `profit-web.conf` 至少包含：

- `data-screen`
- `console/users`
- `console/system`

命令：

```bash
npm run gate:nginx
```

### 4.2 SHA 门禁（同口径）

默认口径必须一致：

- 本地目标 SHA（通常 HEAD）
- `origin/<branch>`
- 服务器 bare 仓 `main`（`/opt/git/profit-web.git`）
- 服务器工作树 HEAD（`/opt/profit-web`）

命令（推荐）：

```bash
npm run gate:sha
```

底层校验口径：

```bash
git --git-dir=/opt/git/profit-web.git rev-parse main
git -C /opt/profit-web rev-parse HEAD
```

### 4.3 失败策略（硬失败）

**SHA 不一致 => 发布失败，禁止交付。**  
必须先修复一致性，再继续后续流程。

```bash

```

等价核心命令：

```bash

```
