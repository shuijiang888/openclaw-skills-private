## 1. 系统简介与价值主张

### 1.1 系统定位

盈利管理系统是面向**先进制造与科技型企业**的定价治理与利润管控平台。系统以"把每一次报价变成可审计的利润决策"为核心理念，统一成本口径、系数规则与分层审批，打通一线报价与管理层盈利视角。

### 1.2 四大核心价值

| 序号 | 价值主张 | 说明 |
|------|----------|------|
| 01 | **分钟级报价响应** | 成本拆解 + 多维系数 + 对标参考，一键生成建议价与毛利区间 |
| 02 | **分层审批与留痕** | 折扣穿透自动匹配审批链，时间线记录议价与决策依据 |
| 03 | **盈利罗盘与预警** | 项目象限归类与对策矩阵，支撑管理层看结构、控风险 |
| 04 | **自动与人机分流** | 低复杂度订单走自动通道，战略单强制人机协同 |

### 1.3 适用场景

系统适用于以下行业场景的报价决策与盈利管理：

- 通信设备
- 汽车电子
- 医疗器械
- 新能源装备
- 半导体封测
- 工业自动化

### 1.4 端到端主流程

```
❶ 需求与产品 → ❷ 物料清单/成本 → ❸ 系数引擎 → ❹ 智能报价 → ❺ 分层审批 → ❻ 罗盘复盘
```

## 2. 快速启动指南

### 2.1 环境要求

- **Node.js** ≥ 18
- **npm** 包管理器
- 推荐浏览器：Chrome、Edge、Firefox 最新版

### 2.2 首次安装与启动

在项目根目录（包含 `package.json` 的文件夹）中执行：

```bash
npm install
npx prisma db push
npm run db:seed          # 全量种子数据（仅空库使用，会清空已有数据）
npm run dev              # 启动开发服务器
```

启动后终端出现 `Ready` 字样即可访问：

- **首选**：http://127.0.0.1:3000
- **备用**：http://localhost:3000

> **注意**：必须保持终端窗口打开，关闭终端会停止服务。

### 2.3 已有业务数据的环境

如果数据库中已有真实业务数据，**请勿**执行全量 `npm run db:seed`，改用安全命令：

```bash
npx prisma db push
npm run db:seed:reference    # 仅补齐罗盘阈值和对策规则，不清空业务数据
npm run dev
```

### 2.4 身份模式说明

系统支持两种身份认证模式：

| 模式 | 特点 | 切换方式 |
|------|------|----------|
| **演示模式**（默认） | 右上角下拉切换「试点角色」，无需登录 | 默认启用，由 `x-demo-role` 请求头驱动 |
| **登录模式**（推荐上线） | 右上角为登录/退出按钮，角色由账号绑定 | 在 `.env` 中设置 `PROFIT_AUTH_MODE=session` 和 `NEXT_PUBLIC_PROFIT_AUTH_MODE=session` |

**登录模式创建管理员账号**：

```bash
npx prisma generate && npx prisma db push
npm run auth:create-user -- '邮箱' '密码' ADMIN
```

### 2.5 健康检查

```bash
curl -s http://127.0.0.1:3000/api/health
```

### 2.6 常见问题排查

| 问题 | 解决方案 |
|------|----------|
| 浏览器显示"无法连接" | 检查是否已运行 `npm run dev` 并等到出现 `Ready` |
| 端口 3000 被占用 | 执行 `npx next dev -p 3001`，浏览器改为 http://127.0.0.1:3001 |
| 盈利罗盘报错/500 | 依次执行 `npx prisma generate`、`npm run db:repair`（或 `npx prisma db push`）、`npm run db:seed:reference`，然后重启 `npm run dev` |
| 页面空白或模块缺失 | 先在终端查看报错信息，常见为未安装依赖（需先 `npm install`） |

## 3. 角色与权限说明

### 3.1 角色总览

系统设有 5 种角色，每种角色拥有不同的功能权限和界面呈现：

| 角色 | 中文名称 | 权限等级 |
|------|----------|----------|
| `SALES_MANAGER` | 销售经理 | 基础操作 |
| `SALES_DIRECTOR` | 销售总监 | 中层管理 |
| `SALES_VP` | 销售副总裁 | 高层管理 |
| `GM` | 总经理 | 最高业务权限 |
| `ADMIN` | 管理员 | 全系统权限 |

### 3.2 功能模块权限矩阵

| 角色 | 门户 | 工作台 | 项目管理 | 新建报价 | 盈利罗盘 | 战略全文 | 路线图 | 管理后台 | AI 助手 |
|------|------|--------|----------|----------|----------|----------|--------|----------|---------|
| 销售经理 | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ | ✗ | ✗ | ✓（含上岗清单） |
| 销售总监 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ | ✓（全功能） |
| 销售副总裁 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ | ✓（全功能） |
| 总经理 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓（只读） | ✓（全功能） |
| 管理员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓（全权限） | ✓（全功能） |

### 3.3 管理后台细分权限

| 后台子页面 | 总经理 | 管理员 |
|-----------|--------|--------|
| 控制台首页 | ✓ | ✓ |
| 项目与审批（Pipeline） | ✓ | ✓ |
| 主数据 · 客户 | ✓（只读） | ✓（含 CSV 导入） |
| 落地准备 | ✓ | ✓ |
| 系数与规则 | ✗ | ✓ |
| 智能体审计 | ✗ | ✓ |

### 3.4 审批权限层级

折扣率决定审批链的层级匹配，各角色的折扣审批权限如下：

| 审批层级 | 折扣权限范围 | 审批条件说明 |
|----------|-------------|-------------|
| 销售经理 | ≤ 5% | 标准范围内的常规折扣 |
| 销售总监 | 5% – 15% | 超出经理权限的中等折扣 |
| 销售副总裁 | 15% – 20% | 大额订单或战略客户折扣 |
| 总经理 | > 20% 或特批 | 特殊项目、亏损风险审批 |

### 3.5 各角色操作建议

**销售经理**：
- 重点关注项目的新建与报价，确保成本录入准确
- 利用 AI 助手的"上岗清单"功能快速熟悉业务流程
- 通过盈利罗盘了解自己负责项目的象限分布

**销售总监**：
- 关注待审批队列中折扣在 5%–15% 范围的项目
- 利用工作台的"经营简报"了解团队整体盈利状况
- 通过参考对比分析指导下属优化报价策略

**销售副总裁**：
- 重点审批大额订单和战略客户的特殊定价
- 利用盈利罗盘监控整体项目组合的健康度
- 关注毛利率低于 15% 的"风险池"项目

**总经理**：
- 通过"经营简报"一站式掌握全局指标：低毛利风险池、自动通道占比、待审批数量、客户集中度
- 在管理后台查看项目管线和客户数据
- 利用盈利罗盘做战略决策与复盘

**管理员**：
- 负责系统主数据的维护和 CSV 批量导入
- 管理系数与审批规则的配置
- 通过智能体审计追踪所有 AI 辅助操作的记录

## 4. 功能模块操作指南

### 4.1 工作台（经营概览）

**访问路径**：`/dashboard`

工作台是登录后的核心操作页面，提供经营数据的全局概览。

#### 4.1.1 角色引导面板

页面顶部根据当前角色显示不同的引导信息和操作建议，帮助用户快速定位最关键的待办事项。

#### 4.1.2 经营简报（Boss Briefing）

经营简报面板以管理层视角呈现关键指标，包含：

| 指标 | 含义 | 关注点 |
|------|------|--------|
| 低毛利风险池 | 毛利率低于 15% 的项目数量 | 红色高亮，需立即关注 |
| 自动通道占比 | 符合自动报价条件的项目比例 | 衡量标准化程度 |
| 待审批数量 | 当前等待审批的项目数 | 审批效率指标 |
| 客户集中度 | 头部客户贡献占比 | 风险分散评估 |

#### 4.1.3 价值指标看板

四张可点击的指标卡片，点击后直接跳转到对应的管理页面：

- **毛利低于 15%**：跳转至项目列表，聚焦风险项目
- **自动通道**：跳转至项目列表，查看可自动化处理的订单
- **待审批**：跳转至审批队列
- **已核准**：跳转至已成交项目

#### 4.1.4 统计卡片

以四宫格形式展示：

- 📋 **项目总数**：跳转至项目列表
- 👥 **客户主数据**：跳转至客户管理
- 🧭 **罗盘项目**：跳转至盈利罗盘
- ⏳ **待审批**：跳转至审批队列

#### 4.1.5 最近项目列表

展示最近更新的 5 个项目，每项显示：
- 项目名称
- 客户名称
- 当前状态
- 建议报价（如已生成）
- 快速入口，点击"进入"直接跳转到报价工作台

> **操作提示**：如果列表为空，点击"新建报价"创建第一个项目，或者在空库环境下执行 `npm run db:seed` 填充示例数据。

#### 4.2.2 新建项目

**访问路径**：`/projects/new`

新建报价页面用于创建项目并自动生成首版建议价与对标参考。

**操作步骤**：

1. **选择客户**
   - 从下拉列表中选择已有客户
   - 或输入新客户名称，系统自动创建（默认评级为"普通客户"）

2. **填写项目详情**

   | 字段 | 说明 | 默认值 |
   |------|------|--------|
   | 项目名称 | 本次报价的项目标题 | "新询价项目" |
   | 产品/规格 | 产品名称或规格型号 | 空 |
   | 数量（PCS） | 预计采购数量 | 1000 |
   | 交付周期（天） | 预计交付天数 | 15 |
   | 标准品 | 是否为标准化产品 | ✓ 勾选 |
   | 小额订单 | 是否为小额订单 | ✓ 勾选 |

3. **录入成本基准**

   | 成本项 | 说明 | 默认值 |
   |--------|------|--------|
   | 直接材料 | 铜箔、树脂、覆铜板等原材料成本 | ¥8,500 |
   | 直接人工 | 钻孔、电镀、检测等工时成本 | ¥1,200 |
   | 制造费用 | 设备折旧、厂租、能耗等间接成本 | ¥2,300 |
   | 期间费用 | 管理、销售、研发分摊费用 | ¥1,000 |

4. **提交创建**
   - 点击"创建项目并进入工作台"按钮
   - 系统自动计算建议价格，生成对标参考
   - 创建成功后自动跳转到报价工作台

> **操作提示**：成本的准确性直接影响后续报价和毛利计算。建议与财务部门确认最新的标准成本数据后再录入。

### 4.3 智能报价工作台（核心）

**访问路径**：`/projects/[id]`

智能报价工作台是系统的核心功能页面，集成了 9 个功能模块，覆盖从成本测算到审批成交的完整报价流程。

#### 模块 ① 项目摘要

位于页面左上方的信息卡片，展示项目基本信息：

| 信息项 | 说明 |
|--------|------|
| 客户名称 | 关联客户 |
| 客户评级 | 战略客户（琥珀标签）、重点客户（蓝色标签）、普通客户（灰色标签） |
| 项目状态 | 草稿 / 已定价 / 待审批 / 已核准（颜色编码） |
| 产品/规格 | 产品型号名称 |
| 数量 / 交期 | 采购数量与交付天数 |
| 标品 / 小额 | 标准品标记和小额订单标记（✓ 或 ✗） |
| 核准成交价 | 审批通过后显示最终成交价格 |

#### 模块 ② 成本基准

位于项目摘要右侧，提供四项成本的可编辑输入和合计展示：

| 成本项 | 对应内容 | 操作说明 |
|--------|----------|----------|
| 直接材料 | 铜箔 + 树脂 + 覆铜板 | 输入框可直接修改，失焦后自动保存 |
| 直接人工 | 钻孔/电镀/检测工时 | 同上 |
| 制造费用 | 设备折旧/厂租/能耗 | 同上 |
| 期间费用 | 管理 + 销售 + 研发 | 同上 |

底部显示**初始成本合计**（四项之和），以醒目的橙色字体呈现。

> **操作提示**：修改任一成本项后，鼠标移出输入框即自动触发后端保存和价格重算。已核准的项目所有输入框均为只读状态。

#### 模块 ③ 智能系数叠加引擎

这是报价计算的核心引擎，通过 6 个维度的系数连乘得出建议报价。

**系数表格**：

| 系数 | 说明 | 标签示例 | 影响方向 |
|------|------|----------|----------|
| 客户系数 | 基于客户评级调整溢价 | "战略客户" / "普通客户" | 战略客户通常 > 1.0（溢价保护） |
| 行业系数 | 行业景气与需求弹性 | "行业适配" | 景气行业上调 |
| 区域系数 | 物流与属地成本差异 | "区域调节" | 偏远区域可能下调 |
| 产品系数 | 产品技术门槛与复杂度 | 产品名称缩写 | 高技术门槛产品上调 |
| 交期系数 | 交付周期紧急程度 | "加急订单" / "常规交期" / "宽裕交期" | 加急订单上调 |
| 批量系数 | 规模效应折扣 | "大批量" / "中批量" / "小批量" | 大批量通常 < 1.0（让利） |

表格每行显示：系数名称、当前数值（可编辑）、标签、影响百分比（+X% 红色表示涨价 / -X% 绿色表示降价）、原因说明。

**雷达图可视化**：以六边形雷达图直观展示各系数的相对强度，颜色为琥珀色。

**建议报价公式**：

```
建议报价 = 初始成本合计 × 客户系数 × 行业系数 × 区域系数 × 产品系数 × 交期系数 × 批量系数
```

公式以文字形式展示在引擎面板中，便于理解和审计。

**建议报价**：以红色渐变背景的大字体卡片展示，带有呼吸式发光动画效果，下方标注对应的建议毛利率。

> **操作提示**：每个系数均可手动微调。修改后鼠标移出输入框会自动重算并同步到后端，同时刷新对标参考数据。

#### 模块 ④ 参考对比分析

以表格形式展示建议价格与多维度参考价格的对比：

| 参考类型 | 说明 |
|----------|------|
| 历史报价 | 同类产品/客户的历史成交价 |
| 竞品价格 | 已知竞争对手的报价水平 |
| 市场价格 | 行业通行的市场价格 |
| 上次成交价 | 与该客户最近一次的成交价 |
| 成本 + 30% | 成本基础上加 30% 的保守定价 |

每项参考价格标注与建议价的**差异百分比**：正值（红色）表示建议价高于参考，负值（绿色）表示低于参考。

> **操作提示**：对比分析帮助销售人员在谈判中有据可依。如果建议价明显高于历史成交价，需准备充分的涨价理由（如工艺升级、材料变更）。

#### 模块 ⑤ 胜率预测模型

综合评估当前报价的中标概率，包含 5 个维度：

| 维度 | 权重 | 说明 | 评估标准 |
|------|------|------|----------|
| 价格竞争力 | 28% | 报价在市场中的竞争力 | ≥80 "高于市场"，<80 "略低" |
| 客户关系 | 22% | 与客户的合作基础 | ≥80 "合作稳定"，<80 "需加强" |
| 交付能力 | 18% | 产能与交期保障 | ≥80 "产能充足"，<80 "需评估" |
| 技术匹配 | 20% | 产品技术与客户需求的契合度 | ≥80 "工艺匹配"，<80 "需确认" |
| 付款条件 | 12% | 付款方式与账期的合理性 | ≥70 "条款合理"，<70 显示具体账期 |

**可视化呈现**：
- **仪表盘图**：以半环形仪表盘展示综合胜率百分比
- **进度条**：每个维度以彩色进度条展示分值（绿色 ≥80、琥珀色 60-79、红色 <60）
- **可编辑分值**：每个维度的分值均可手动调整，失焦后自动保存

> **操作提示**：胜率预测为参考值，帮助团队判断是否需要在谈判策略上做调整。建议在客户拜访前检查各维度分值，针对性准备应对方案。

#### 模块 ⑥ 客户还价与折扣计算

用于录入客户还价并自动计算折扣率和审批需求。

**操作步骤**：

1. 在"还价金额"输入框中输入客户的实际还价（留空表示未还价）
2. 系统自动计算并显示：
   - **折扣率**：还价相对建议价的折扣百分比
   - **还价后毛利率**：以还价计算的毛利率（低于 15% 显示红色警告，≥15% 显示绿色）
3. **建议审批链**：根据折扣率自动匹配审批层级（如"销售总监（5%–15%）"）

> **操作提示**：填入还价后，下方会即时显示推荐的审批角色。如果折扣率较高（>15%），建议在提交审批前准备详细的让利理由。

#### 模块 ⑦ 授权审批规则

以表格形式展示四级审批规则，当前匹配的层级会以 ▶ 标记和高亮底色标识：

| 层级 | 折扣权限 | 审批条件 |
|------|----------|----------|
| 销售经理 | 0% – 5%（含） | 标准范围内 |
| 销售总监 | >5% – 15%（含） | 超出经理权限 |
| 销售副总裁 | >15% – 20%（含） | 大额订单/战略客户 |
| 总经理 / 特批 | >20% | 特殊项目/亏损风险 |

#### 模块 ⑧ 智能决策分流

以双通道对比的形式展示报价分流结果，当前命中的通道会以彩色边框和"✓ 当前"标签突出显示。

**自动报价通道**（绿色高亮时为命中）：

需同时满足全部条件：
- 产品类型 = 标品/标准型号
- 订单规模 = 小项目/小订单
- 折扣范围 ≤ 5%
- 客户评级 = 普通客户
- 毛利率 ≥ 25%

策略："报不到就拿不到"——标准化快速响应。

**人机协同通道**（琥珀色高亮时为命中）：

任一条件满足即触发：
- 产品类型 = 定制化/复杂产品
- 客户评级 = 战略/大客户
- 订单规模 = 大项目/大批量
- 折扣范围 > 5%（超权限）
- 毛利率 < 25%

策略："不惜代价拿下"——确保重点项目不流失。

底部显示本单具体命中的条件明细。

> **操作提示**：分流结果会影响后续的审批流程。自动通道的订单审批流程更短，人机协同的订单需要更高层级的审批把关。

#### 模块 ⑨ 智能报价单

以三栏布局呈现报价单的完整信息：

**左栏 — 报价信息**：
- 客户名称、项目名称、产品规格
- 数量（PCS）、交付周期
- 初始成本、系数调整系数
- **建议报价**（红色加粗）
- 建议毛利率

**中栏 — AI 报价建议**：
- 当前毛利率分析
- 基于客户评级的议价策略建议
- 价格变动时的沟通建议

**右栏 — 报价状态追踪**：
- 时间线事件列表（最近 5 条）
- 每条事件带有状态点：绿色（已完成）、琥珀色（待处理）、蓝色（进行中）
- 如有待审批，底部显示闪烁的"→ 待审批"状态和审批角色

**操作按钮**（位于底部）：

| 按钮 | 功能 | 可用条件 |
|------|------|----------|
| **提交审批** | 将报价提交到审批流 | 项目未锁定且无待审批 |
| **重新测算** | 重新计算建议价和刷新对标 | 项目未锁定 |
| **审批通过** | 审批通过并锁定成交价 | 有待审批且当前角色权限不低于所需 |

审批提示：如果当前角色无权审批，页面会提示"请切换为不低于该权限的角色后再审批"。

**其他功能**：
- **复制报价摘要**：页面顶部的按钮，一键将报价详情复制到剪贴板，方便邮件或沟通使用
- **请求 ID 追踪**：每次保存操作会显示请求 ID，可在"智能体审计"页面中检索核查

**访问路径**：`/compass`

盈利罗盘是面向管理层的经营分析工具，基于**毛利率 × 增长率**双维度将项目分入四象限，辅以评估指标和对策矩阵。

#### 4.4.1 四象限散点图

页面主区域展示 SVG 散点图：

| 左上 | 💰 现金牛 | 高毛利 · 低增长 | 蓝色 |
| 右下 | ❓ 问题项目 | 低毛利 · 高增长 | 琥珀色 |
| 左下 | 🐕 瘦狗项目 | 低毛利 · 低增长 | 红色 |

**交互操作**：
- **悬停气泡**：显示项目名称、客户、毛利率、增长率等详细信息
- **虚线阈值**：毛利率阈值（默认 ≥20% 为高毛利）和增长率阈值（默认 ≥15% 为高增长）
- 图例在右上角，标注四种颜色对应的象限

#### 4.4.2 象限分布环形图

右侧以甜甜圈图展示各象限的项目数量分布：
- 中心显示项目总数
- 四段弧形分别对应四个象限
- 下方为象限策略摘要卡片，点击可跳转项目列表

| 象限 | 策略 | 具体行动 |
|------|------|----------|

五维度评估体系，每个指标分三档：

## 5. AI 智能助手使用指南

### 5.1 概述

报价工作台右侧集成了 AI 智能助手面板，支持本地大模型（Ollama qwen3.5:35b）的自然语言解析能力，帮助用户通过对话方式调整报价系数。

### 5.2 启用与配置

#### 默认状态（规则引擎）

未配置 Ollama 时，助手仅使用内置规则引擎进行解析，无需任何额外配置。

#### 启用本地大模型

1. **安装 Ollama**：从 https://ollama.com/ 下载安装

2. **拉取模型**：
   ```bash
   ollama pull qwen3.5:35b
   ```

3. **配置环境变量**（在 `.env` 文件中添加）：
   ```bash
   OLLAMA_ENABLED=1
   OLLAMA_MODEL=qwen3.5:35b
   # OLLAMA_BASE_URL=http://127.0.0.1:11434    # 非默认端口时配置
   OLLAMA_TIMEOUT_MS=300000                      # 超时设置，35B 模型建议不低于 300 秒
   # OLLAMA_JSON_FORMAT=0                        # 若 Ollama 版本不支持 format:json 时关闭
   ```

4. **重启开发服务器**：`npm run dev`

5. **验证连接**：打开任意项目工作台 → 助手面板 → 点击"检测连接"
   - 显示"服务可达，模型已就绪"即为成功

### 5.3 功能说明

#### 5.3.1 角色感知

AI 助手根据当前登录角色提供差异化的提示和快捷操作：

| 角色 | 助手特性 |
|------|----------|
| 销售经理 | 侧栏显示"上岗清单"辅助指引，提供新手操作提示 |
| 销售总监及以上 | 全功能开放：示例、角色话术追加、快捷操作 |

#### 5.3.2 自然语言解析

在助手输入框中输入中文商机描述，AI 会自动解析并转换为系数调整建议。

**使用示例**：

> 输入："客户是新能源行业的战略大客户，这次是加急订单，7天要交付，量比较大有5万件"
>
> AI 解析结果：建议将客户系数调至 1.20、行业系数 1.15、交期系数 1.12、批量系数 0.88……

解析完成后，点击"应用"即可自动将系数调整写入工作台。

#### 5.3.3 快捷操作

助手面板提供以下快捷操作按钮：

| 操作 | 功能 |
|------|------|
| 重新测算 | 触发工作台的价格重算 |
| 提交审批 | 一键提交当前报价到审批流 |
| 审批通过 | 审批通过（需权限） |
| 刷新项目 | 重新加载项目最新数据 |

#### 5.3.4 回退机制

- Ollama 服务不可达时，自动回退到内置规则引擎
- 可随时点击"仅用规则"按钮手动切换
- 接口超时由 `OLLAMA_TIMEOUT_MS` 控制，默认 5 分钟

### 5.4 接口说明

| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/assistant/ollama-status` | GET | 检测 Ollama 服务状态与模型可用性 |
| `/api/assistant/quote-parse` | POST | 发送自然语言描述，获取系数调整建议 |

> **操作提示**：首次使用 35B 模型推理可能需要数十秒到数分钟（取决于 GPU/CPU 性能）。如果等待时间过长，可适当降低模型规格或增加 `OLLAMA_TIMEOUT_MS` 的值。

## 6. 核心业务流程（端到端）

### 6.1 从客户需求到报价生成

```
┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│ 1.接收需求 │ →  │ 2.创建项目 │ →  │ 3.录入成本 │ →  │ 4.系数引擎 │
└───────────┘    └───────────┘    └───────────┘    └───────────┘
                                                         │
                                                         ▼
                                                  ┌───────────┐
                                                  │ 5.生成报价 │
                                                  └───────────┘
```

**详细步骤**：

1. **接收客户需求**
   - 获取客户的产品需求、数量、交期等信息
   - 确认客户在系统中的评级（战略/重点/普通）

2. **创建项目**（`/projects/new`）
   - 选择或新建客户
   - 填写项目名称、产品规格、数量、交付周期
   - 勾选"标准品"和"小额订单"标记（影响分流结果）

3. **录入成本基准**
   - 录入四项成本：直接材料、直接人工、制造费用、期间费用
   - 建议与财务部门确认最新标准成本

4. **系数引擎计算**
   - 系统自动加载默认系数
   - 根据客户评级、行业、区域、产品特性等自动调整
   - 可手动微调任一系数，或使用 AI 助手自然语言调整

5. **生成建议报价**
   - 系统自动计算：建议报价 = 成本合计 × 系数连乘积
   - 同步生成：参考对比数据、胜率预测、分流建议

### 6.2 从报价到审批

```
┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│ 1.客户谈判 │ →  │ 2.录入还价 │ →  │ 3.确认折扣 │ →  │ 4.提交审批 │
└───────────┘    └───────────┘    └───────────┘    └───────────┘
```

**详细步骤**：

1. **客户谈判**
   - 将建议报价提供给客户（可使用"复制报价摘要"功能）
   - 参考"参考对比分析"准备谈判策略
   - 参考"胜率预测"评估中标概率

2. **录入客户还价**
   - 在模块 ⑥ "客户还价与折扣计算"中输入客户的还价金额
   - 系统自动计算折扣率和还价后毛利率

3. **确认折扣与审批链**
   - 查看折扣率对应的审批层级
   - 如折扣过大（毛利率 < 15%），系统会以红色警示
   - 确认是否需要更高层级的审批

4. **提交审批**
   - 点击"提交审批"按钮
   - 系统自动将项目路由到对应层级的审批人
   - 项目状态变为"待审批"
   - 审批操作留痕，可在审计页面追溯

### 6.3 从审批到成交

```
┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│ 1.审批通知 │ →  │ 2.查看报价 │ →  │ 3.审批决策 │ →  │ 4.成交锁定 │
└───────────┘    └───────────┘    └───────────┘    └───────────┘
```

**详细步骤**：

1. **审批通知**
   - 审批人在工作台或项目列表"我的队列"中看到待审批任务
   - 管理后台"待审批队列"也会显示

2. **查看报价详情**
   - 点击项目进入工作台，查看完整的报价数据
   - 审核成本基准、系数设置、参考对比、胜率预测
   - 检查分流结果和折扣率

3. **审批决策**
   - 确认当前角色具有审批权限（页面会提示）
   - 如权限不足，在演示模式下切换至更高角色；登录模式下联系有权限的审批人
   - 点击"审批通过"按钮完成审批

4. **成交锁定**
   - 审批通过后，项目状态变为"已核准"
   - 核准成交价锁定显示在项目摘要中
   - 所有输入框变为只读，防止误修改
   - 审批时间线完整记录，支持复盘

> **重要提示**：审批通过后项目即锁定，无法再修改。如需调整，需要联系管理员在后台处理。

## 7. 附录：系统核心价值指标

### 7.1 关键业务指标（KBI）

| 指标 | 计算公式 | 健康阈值 | 说明 |
|------|----------|----------|------|
| 建议毛利率 | (建议价 - 成本) / 建议价 × 100% | ≥ 25% | 低于 15% 进入风险池 |
| 还价后毛利率 | (还价 - 成本) / 还价 × 100% | ≥ 15% | 低于 15% 红色警告 |
| 折扣率 | 1 - 还价 / 建议价 | ≤ 5% 自动审批 | 超过 20% 需总经理特批 |
| 综合胜率 | 五维加权评分 | ≥ 70% | 低于 60% 需调整策略 |
| 自动通道占比 | 自动通道项目数 / 总项目数 | 越高越好 | 衡量标准化程度 |
| 客户集中度 | 头部客户收入 / 总收入 | ≤ 30% | 超过 40% 为高风险 |

### 7.2 系数说明与默认值

| 系数 | 默认值 | 影响方向 | 说明 |
|------|--------|----------|------|
| 客户系数 | 1.15 | 溢价保护 | 战略客户 > 1.0，利于长期合作 |
| 行业系数 | 1.10 | 行情调整 | 景气行业上浮 |
| 区域系数 | 0.95 | 物流适配 | 偏远区域下调 |
| 产品系数 | 1.20 | 技术门槛 | 高技术产品上调 |
| 交期系数 | 1.08 | 紧急溢价 | 加急订单上调 |
| 批量系数 | 0.92 | 规模让利 | 大批量下调 |

**建议报价公式**：

```
建议报价 = (直接材料 + 直接人工 + 制造费用 + 期间费用) × 客户系数 × 行业系数 × 区域系数 × 产品系数 × 交期系数 × 批量系数
```

### 7.3 胜率权重模型

| 维度 | 权重 | 说明 |
|------|------|------|
| 价格竞争力 | 28% | 报价在市场中的相对位置 |
| 客户关系 | 22% | 合作历史与信任基础 |
| 技术匹配 | 20% | 产品技术与客户需求契合度 |
| 交付能力 | 18% | 产能保障与交期履约能力 |
| 付款条件 | 12% | 账期与付款方式的灵活度 |

**综合胜率** = Σ (各维度分值 × 对应权重)，结果限制在 0–100% 之间。

### 7.4 罗盘四象限划分

| 象限 | 毛利率条件 | 增长率条件 | 策略 |
|------|-----------|-----------|------|
| 明星项目 | ≥ 阈值（默认 20%） | ≥ 阈值（默认 15%） | 加大投入，重点保护 |
| 现金牛 | ≥ 阈值 | < 阈值 | 维持现状，稳定收益 |
| 问题项目 | < 阈值 | ≥ 阈值 | 优化成本或战略性放弃 |
| 瘦狗项目 | < 阈值 | < 阈值 | 逐步退出，释放资源 |

阈值可在管理后台 `/console/rules` 页面自定义配置。

### 7.5 CSV 导入模板汇总

| 模板文件 | 导入入口 | 适用场景 |
|----------|----------|----------|
| `/templates/customers-import-template.csv` | 管理后台 · 主数据 · 客户 | 批量创建客户 |
| `/templates/projects-import-template.csv` | 管理后台 · 项目与审批 | 批量创建项目与报价 |

> 表头大小写不敏感。所有 CSV 导入操作均需管理员权限。
