### Step 2：实现问卷部分

**问卷结构（26题）：**

| 章节 | 题号 | 类型 | 说明 |
|------|------|------|------|
| 企业基本信息 | Q1-Q4 | 单选+填空 | 企业名称/行业/规模等 |
| D1市场定位 | Q5-Q8 | 单选+多选 | 含Q5多选计分 |
| D2获客渠道 | Q9-Q12 | 单选+多选 | 含Q12反向计分 |
| D3销售管道 | Q13-Q16 | 单选+多选 | 含Q15/Q16计分 |

| 联系信息 | Q25-Q26 | 填空+开放 | 仅用于线索收集 |

**交互要求：**
- 每个维度前有简短说明（1-2句）
- 进度条显示当前维度/总维度
- 选项点击后立即高亮选中
- 多选题选项可切换（选中/取消）
- 必填项未答时点击"下一题"要提示

### Step 3：实现计分引擎（JavaScript）

**必须实现的计分规则：**

```javascript
// 单选：直接取选项分值（1-5）

// Q5多选：得分 = min(5, N+1)，N为前6项选中数；仅勾"无渠道"得1分
// Q8反向：得分 = max(0, 5 - 勾选数)
// Q11起始5分：勾"产品不匹配/关系失守/协同慢"各-1；勾"未总结"再-1；最低0
// Q15反向：得分 = max(0, 5 - 勾选数)
// Q19反向：得分 = max(0, 5 - 勾选数)
// Q23多选：N=0→0分；N=1→3分；N=2→5分；N≥3→5分

// 维度加权：
// D1(市场定位) = 20分，D2(获客) = 20分，D3(管道) = 20分
// D4(客户成功) = 15分，D5(团队) = 15分，D6(数字化) = 10分
// 综合分 = D1+D2+D3+D4+D5+D6（满分100）
```

**等级判定：**
- 90-100：卓越
- 70-89：良好
- 50-69：一般
- 30-49：薄弱
- 0-29：危机

### Step 4：实现报告摘要展示

提交后：
1. 隐藏问卷，显示报告摘要
2. 显示综合得分 + 等级
3. 显示6个维度得分（文字列表）
4. 显示TOP3核心问题（根据维度得分最低的3个）
5. 底部CTA："截图保存 / 预约专家解读 / 重新填写"
6. 联系方式表单（姓名+手机+公司）— 收集线索
