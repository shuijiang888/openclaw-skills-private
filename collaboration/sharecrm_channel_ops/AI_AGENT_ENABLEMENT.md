# AI Agent 在纷享销客渠道运营场景中的赋能与价值最大化

本文在**现有原型能力**（渠道主数据、情报、预警、作战台、绩效、报价、导入审计、渠道 360° 三流叙事）之上，说明 **对象关联与下钻**如何支撑人机协同，以及 **AI Agent** 按角色、按场景落地时可带来的增量价值与边界。

---

## 1. 数据对象与必要关联（系统内语义）

| 对象 | 主键/锚点 | 与谁强关联 | 业务含义 |
|------|-----------|------------|----------|
| 渠道 `channel` | `id`、`channel_code`、`country_code` | 预警、月度出货、负责人、竞品/活动、情报国别 | 一线作业与回款的主实体 |
| 国别情报 `market_intel` | `country_code` | 同国渠道集合、销售简报、报价国家维度 | 「看见市场」与资源排序 |
| 预警 `alert` | `id` → `channel_id` | 渠道、作战台 SLA、绩效流程灯 | 风险闭环与晨会清单 |
| 导入批次 `import_batch` | `id` | 操作者、时间、成功/失败行 | 治理与审计，支撑「谁改了什么」 |
| 渠道 360 子对象 | `channel_competitor`、`channel_activity` | 单渠道 | 竞争与过程工单，反哺情报与业务流叙事 |

**价值最大化前提：** 任意界面从「汇总」跳到「可行动的单据」时，应携带 **country / channel_id / region** 之一，避免用户重新筛选。原型中已通过图表点击、表格按钮、元数据快捷链等体现该原则。

---

## 2. 典型下钻路径（与界面行为对齐）

- **总览 → 区域柱图** → 渠道列表按 **区域** 筛选。  
- **总览 → A/B/C 环形图** → 渠道列表按 **分级** 筛选。  
- **总览 → TOP 渠道横条** → **渠道详情**（含 360° 三流）。  
- **总览 → 国别机会 TOP** → **该国情报详情**（并可继续跳到该国渠道、报价）。  
- **预警中心** → 渠道编码链接 / 「渠道详情」→ 单渠道闭环。  
- **绩效 · 区域表** → 「区域渠道」→ 列表筛选。  
- **绩效 · 关注清单** → 「渠道详情」→ 逾期与 CLV 上下文。  
- **作战台 · 覆盖缺口** → 一键进入渠道列表或情报总表。  
- **渠道详情页眉** → 情报 / 该国渠道 / 本区域 / 报价 / 作战台 / 预警中心。

这些路径把 **信息流（情报）— 业务流（预警/工单）— 资金流（出货/毛利/应收）** 串成可演示、可扩展的 **导航图**；AI Agent 应能生成/理解同一套实体 ID 与跳转意图。

## 3. 按角色的 Agent 赋能（场景 × 价值）

### 3.1 销售（一线，范围：本人负责渠道）

| 场景 | 人类痛点 | Agent 可做什么 | 价值 |
|------|----------|----------------|------|
| 拜访前 10 分钟 | 信息散在情报、备注、活动里 | 拉取 `channel_360` + 该国 `intel` + 未关 `alert`，生成**一页拜访摘要**与 3 条建议话术 | 减少漏项，提高当面转化率 |
| 周报/回填 | 不愿写系统 | 根据活动类型与时间线**草稿纪要**，人改后写入 `channel_activity` 或备注 | 主数据与过程数据同步 |
| 报价沟通 | 规则计算器与商务条款分离 | 结合 `POST /tools/quote` 结果，生成**给客户的话术**（不含虚构折扣承诺） | 加速响应，边界清晰 |

### 3.2 渠道总监 / 运营（manager，全局或大范围）

| 场景 | 人类痛点 | Agent 可做什么 | 价值 |
|------|----------|----------------|------|
| 晨会 | 作战台数字多、难叙事 | 读取 `playbook` + `scorecard`，按 SLA 库龄与关注清单**自动生成会议议程**与点名顺序 | 会程缩短、动作可执行 |
| 资源投放 | 不知优先哪国、哪类渠道 | 用 `intel.opportunity_score` × 渠道分级 × 预警密度做**排序解释**（可解释的规则 + 自然语言） | 对齐「情报—渠道—钱」 |
| 导入治理 | 批次多、错误难讲清 | 对 `import_batch` 与 preview 结果做**错误归类与修复建议**（字段级） | 降低二次导入成本 |

### 3.3 高管 / 业务负责人（读多写少）

| 场景 | 人类痛点 | Agent 可做什么 | 价值 |
|------|----------|----------------|------|
| 周/月复盘 | 只看数字不懂故事 | 基于 BSC 四象限 + 区域达成 + 预警趋势，输出**短文叙事**（与「复制高管简报」互补） | 决策语言统一 |
| 风险升级 | Critical 预警分散 | 聚合未关 critical、高逾期、长期未联系，**单页风险摘要** | 减少信息遗漏 |

### 3.4 管理员 / 数据岗（admin）

| 场景 | Agent 可做什么 | 价值 |
|------|----------------|------|
| 模板与校验 | 解释 `import/template` 中字段约束，把客户 Excel **映射**到 JSON 列 | 减少来回沟通 |
| 种子与演示 | 生成符合编码规则的**脱敏演示数据**草案（需人审后导入） | 加速 PoC |

## 4. 实现形态建议（从原型到生产）

1. **Tool-calling 对齐 API**：Agent 仅通过已有 `GET/PATCH/POST` 读写，与前端同源；禁止杜撰 `channel_id`。  
2. **会话上下文**：携带 `user.role` 与 scope（销售仅本人渠道），与后端 `channelScopeWhere` 一致。  
3. **可审计**：写操作走 API，自然留下时间戳与操作者；重要结论引用 **对象 ID / 国家码 / 时间**。  
4. **人机分工**：Agent 产出 **草稿**（纪要、简报、排序理由）；**确认**与合规承诺必须由人完成。  
5. **与「三流」对齐**：回答结构建议固定为 **信息层 / 业务动作层 / 资金与风险层**，与 `channel_360.flows` 叙事一致，便于培训和验收。

---

## 5. 边界与红线

- 不代替 **财务确权、合同条款、法务合规** 结论。  
- 不基于未接入系统内的数据**编造**出货或回款。  
- 对外发送客户可见内容前**必须人工审核**（演示环境亦建议养成习惯）。  
- 生产环境：**密钥、PII、日志**须按公司安全策略托管，Agent 不长期存全量客户明细于提示词。

---

## 6. 与当前原型的衔接

- **下钻与关联**：见前端「总览 / 绩效 / 预警 / 渠道详情」交互；API 侧 `analytics/overview` 的 `topChannels.id`、`scorecard.watchlist.channel_id` 等为下钻提供主键。  
- **Agent 落地第一步**：可选接 OpenAPI 描述或 MCP 封装 `v1/*`，使 Agent 能调用与界面相同的能力，再叠加 RAG（情报正文、简报 Markdown）增强「看见」层。
