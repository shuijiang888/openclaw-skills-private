# AI 诊断建议设计方案（医疗器械版）

> 目标：根据 **六维加权分 + 综合分 + 等级**，生成 **3 条可执行** 的改进建议，避免泛泛而谈。  
> 与 `questionnaire_medical.md`、`benchmark_data.md` 配套；**生产环境建议走后端**，避免密钥暴露与 CORS。

---

## 1. 目标与原则

1. **针对性**：优先覆盖 **得分最低的 1～2 个维度**。  
2. **可执行**：每条含 **动作 + 责任角色 + 时间尺度（30/90 天）**。  
3. **行业性**：至少一条体现 **医疗器械**（集采、入院、合规、经销商、学术）。  
4. **合规**：输出底部 **必须** 带免责声明；**不**输出具体患者数据、未公开价格。

---

## 2. 技术方案

### 方案 A：纯前端 MVP（HANDOFF 默认）

| 项 | 说明 |
|----|------|
| 调用 | 浏览器 `fetch` → KIMI / OpenAI 兼容接口 / Gemini（视 CORS 与密钥策略） |
| 风险 | **API Key 暴露**；多数云厂商 API **禁止浏览器直连** |
| 适用 | 内测、演示、VPN 环境下 PoC |

**缓解：** 使用 **仅推理权限的子 Key** + 域名白名单（若平台支持）+ **脱敏**（不传公司全称、手机，仅传分数与维度标签）。

### 方案 B：生产推荐（后端代理）

| 项 | 说明 |
|----|------|
| 调用 | Node.js `POST /api/v1/submissions/:id/ai-advice` 或独立 `llm-proxy` |
| 缓存 | 同一 `submission_id` + 分数快照 **30 分钟** Redis 缓存 |
| 密钥 | 仅存服务端环境变量；审计日志不落用户 PII |

## 3. Prompt 模板（医疗器械）

```
你是一位医疗器械行业营销与准入顾问。根据以下企业自评得分，输出 3 条「最优先改进建议」：

- 综合得分：{total}/100（等级：{level}）
- D1 市场定位（加权/20）：{d1w}
- D2 获客渠道（加权/20）：{d2w}
- D3 销售管道（加权/20）：{d3w}
- D4 客户成功（加权/15）：{d4w}
- D5 团队激励（加权/15）：{d5w}
- D6 数字化（加权/10）：{d6w}

要求：
1. 每条建议对应一个得分偏低维度，标题用「维度：」开头。
2. 每条包含：具体动作、负责角色（销售/市场/医学/渠道）、建议周期（30 天或 90 天内）。
3. 至少一条必须涉及「集采、入院、招投标、经销商管理、学术推广、合规」中的至少一个主题。
4. 每条举一个行业常见做法示例（可匿名）。
5. 总字数不超过 350 字，使用简体中文，分三条编号输出。
6. 不要索取或假设任何患者隐私或未公开的商务条款。
```

**系统角色（可选）：**  
`你是资深医疗器械行业顾问，回答简洁、可落地，拒绝医疗诊断与投资建议。`

## 4. API 配置（CONFIG 扩展）

```javascript
var CONFIG = {
  // ... 现有 bookingUrl / hotline / reportTitle ...
  LLM_API_KEY: '',  // 占位，勿提交仓库；销售/运维注入
  LLM_PROVIDER: 'kimi',  // 'kimi' | 'openai-compatible' | 'gemini'
  LLM_API_URL: 'https://api.moonshot.cn/v1/chat/completions',
  LLM_MODEL: 'moonshot-v1-8k'
};
```

**KIMI 示例 body：**

```json
{
  "model": "moonshot-v1-8k",
  "messages": [
    { "role": "system", "content": "你是医疗器械行业营销顾问。" },
    { "role": "user", "content": "<prompt>" }
  ],
  "temperature": 0.5,
  "max_tokens": 800
}
```

**Gemini：** 若使用 Google AI Studio，需注意 **区域政策与 key 限制**；建议仍走 **后端**。

---

## 5. 前端交互（与 H5 一致）

- 按钮文案：**「🤖 获取 AI 改进建议」**  
- 点击后：按钮 `disabled`，文案 **「正在生成个性化建议，请稍候…」**  
- 成功：在 `#ai-advice-content` 渲染 Markdown 子集（换行、**粗体**、列表）  
- 失败：提示 **「生成失败，请稍后重试」**；可提供 **离线模板**（按最低维度输出静态三条）  
- 再次点击：**「重新生成」**（可带不同 `temperature` 或加一句「请换角度」）

---

## 6. 免责声明（必须展示）

```
⚠️ AI 建议基于行业通用经验与自评分数生成，不构成医疗、投资或法律意见；具体方案请结合贵司实际并咨询专业人士。
```

---

## 7. 安全与合规

- 不向模型发送 **手机号、完整公司名**（可用「某器械企业」+ 分数）。  
- 记录 **prompt token 用量** 便于成本监控。  
- 大陆部署优先选 **合规备案** 的云模型服务。

## 8. 测试用例（后端单测建议）

|------|----------|------|
