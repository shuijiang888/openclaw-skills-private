## 1. 执行摘要

陈玮体系是一套 **2B 通用**、按 **销售模式 × 测评角色** 分发的 **能力自评问卷 + 维度计分 + 分级研判 + 培训建议** 闭环。与当前仓库中「行业定制 H5（医械/能源/智造）」相比：**陈玮侧更重「销售模式切片」与「多角色（个人 / 管理 / 体系）」；行业 H5 更重「细分行业语境与轻诊断转化」**。

**建议的产品形态：**  
- **统一诊断平台**：问卷内容数据化（JSON/DB），由 **问卷引擎** 按 `role + sale_mode` 渲染；  
- **统一计分服务**：维度满分、题目分值、选项映射可配置；  
- **统一报告服务**：维度得分 → C/B/A 档文案 + 培训课映射（可扩展 LLM 润色，但 **默认以 Excel 规则为真源**）；  
- **组织级视图**（二期）：同一企业下多角色测评结果 **聚合、冲突消解、对标**。

## 2. 源表逻辑还原

### 2.1 销售模式（四象限）

| 模式代码 | Excel 名称 | 说明（来自表头语义） |
|----------|------------|----------------------|
| `raw_material` | 原材料 | 偏大宗/上游，维度「商机培育、竞标、谈判、部分竞争策略」等弱化或不出现在该模式卷中 |
| `semi_custom` | 半成品/定制件 | 题目与分值与「单品」等有差异，维度更全 |
| `standard_product` | 单品/标品 | 与定制件类似但各维度单题分值权重不同 |
| `solution` | 解决方案 | **独有「商机培育」** 维度（10 题），项目运作等齐全 |

**问卷结构 Sheet** 用 √ 标明各模式下 **是否测评该维度**；并给出 **各维度在模式下的满分** 与 **题目数量**，且 **四种模式总分均为 100**。

### 2.2 角色（Persona）

| 角色 | 文件 | 维度集合 | 题量（结构 Sheet） |
|------|------|----------|---------------------|
| 销售人员 | 1.1 | 10 维能力（行业理解、客户获取…项目运作） | 依模式：如原材料合计 50 题，解决方案 100 题等 |
| 销售管理者 | 2.1 | 市场管理、策略管理、流程管理、团队管理、激励管理、赋能培养 | 各维度 5 题 × 6 = **30 题**（ALL）；每模式列合并为 ALL |
| 营销体系 | 3.1 | 市场战略、销售管理、营销流程、营销工具、营销组织、营销激励 | 7+7+7+7+6+6 = **40 题**（ALL） |

### 2.3 题型与选项

各 Sheet 统一为 **三档自评**（列名）：

- **能够做到** / **基本做到** / **有难度**

对应分值 **随维度、模式、角色变化**（示例）：

- 销售人员·原材料·行业理解：**2 / 1.6 / 0.8**（每题，10 题该维满分 20）  
- 销售管理者·市场管理：**4 / 3.2 / 1.6**（每题，5 题该维满分 20）  
- 营销体系·多数维度：**2.5 / 2 / 1**（每题，7 题约 17.5）

**计分规则（还原）：**

- 单题得分 = 所选列对应分值；  
- 维度得分 = 该维度下各题得分 **求和**（且结构表保证不超过该维满分）；  
- 总分 = 各维度得分之和 = **100**。

### 2.4 研判与报告（营销管理者库）

`2.3测评结果数据库（营销管理者）.xlsx`：

- **分级：** C 级 **30–59**、B 级 **60–79**、A 级 **80–100**（表头明示）。  
- **输出结构（每维度）：**  
  - C/B/A 各自：**能力评价**、**能力提升建议**、**培训建议**（课程名列表）。  
- **全局默认句：** 占位「您的测评得分为：XX分，等级为X级，高于/低于行业平均水平」（**行业基准需数据校准**，见 §6）。  
- **培训课程汇总 Sheet：** 高级训战营「基础课 / 扩展课」清单，可与 CRM 或 LMS 对接。

**缺口：** 当前文件中 **未见「销售人员」「营销体系」的独立研判 Sheet**。上线前需陈玮方补齐，或产品侧约定 **暂共用管理者话术模板 / 由顾问人工解读**。

## 3. 信息架构（IA）与关键用户旅程

### 3.1 入口参数（建议）

- `role`：`sales_rep` | `sales_manager` | `marketing_system`  
- `sale_mode`：`raw_material` | `semi_custom` | `standard_product` | `solution`  
- `org_id` / `campaign_id`（可选，用于企业包与渠道归因）

### 3.2 旅程

1. 选择/确认 **销售模式**（可结合 QB：行业、产品线描述辅助默认推荐）。  
2. 选择 **角色**（或邀请链接预填）。  
3. 分维度答题（进度条 + 断点续答）。  
4. 提交 → **即时算分** → 报告页（维度雷达/条形 + 分级文案 + 推荐课程）。  
5. （可选）导出 PDF / 推送顾问 / 写入 CRM。

## 4. 数据模型（逻辑）

### 4.1 核心实体

- **QuestionnaireVersion**：版本号、生效日期、作者。  
- **SaleMode**：四模式定义。  
- **RoleTemplate**：角色 → 维度列表 → 每维度满分、题数。  
- **Dimension**：维度 id、名称、排序、适用 `(role, sale_mode)`。  
- **Question**：题干、所属维度、`role`、`sale_mode`、题型 `likert_3`、选项键 `can / mostly / hard`。  
- **QuestionScoreMap**：`(question_id, option_key) → numeric score`（来自 Excel 各表头行）。  
- **Submission**：提交实例、答题明细、各维得分、总分、等级、时间戳。  
- **ReportSnippet**：`(role, dimension, grade_band)` → 评价/建议/课程（来自 2.3 类库）。  
- **TrainingCourse**：课程名、级别、标签，供 LMS 映射。

### 4.2 与 Excel 的映射策略

- **销售人员**：按 Sheet 名 `原材料` / `半成品&定制件` / `单品&标品` / `解决方案` 各生成一份 **题目包**；与 `问卷结构` 中维度启用矩阵 **交叉校验**（防止漏题/多分）。  
- **管理者 / 体系**：当前为单一 `ALL` Sheet，**四模式共用同一题本**（结构表 ALL 列）；若未来拆分模式，再版本化。

## 5. 计分与等级引擎（可配置）

### 5.1 伪代码

```
total = 0
for each dimension D in active_dimensions(role, sale_mode):
    dim_sum = 0
    for each question Q in D:
        dim_sum += score_table[Q.id][user_choice(Q)]
    assert dim_sum <= D.max_points  // 开发态校验
    total += dim_sum
grade = band(total)  // 全卷 C/B/A 阈值是否因角色而异：需确认；当前仅见管理者 30/60/80
```

## 6. 专业补强与风险（建议在评审中明确）

### 6.1 信度、效度与用途边界

- 本体系为 **自评**，易受 **社会赞许性** 与 **认知偏差** 影响；报告页应声明：**用于发展性诊断与培训规划，不宜作为唯一绩效考核依据**。  
- 建议积累样本后做 **分维度 Cronbach α / 项目区分度** 分析，迭代删改极端项。

### 6.2 合规与表述

- 部分「竞争策略」题干涉及 **收集竞争对手负面信息并在适当时机传递**——在正式产品中建议改为 **「客观对比 / 差异化价值陈述」**，避免违反反不正当竞争法或客户侧合规红线；**需法务或陈玮方修订题干**。  

### 6.3 多角色聚合（组织级）

- 同一企业 **销售管理者得分高 + 一线销售得分低** 可能表明 **体系与执行脱节**；建议二期输出 **组织健康度交叉矩阵**（非简单平均）。  

### 6.4 与行业版 H5 的关系

| 能力 | 陈玮通用 | 现有行业 H5 |
|------|-----------|-------------|
| 销售模式四象限 | ✅ 强 | ❌ 无 |
| 多角色 | ✅ | 单用户自评为主 |
| 行业语境 | 通用 2B | 医械/能源/智造 文案 |
| 轻诊断→完整版 | 未在 Excel 体现 | ✅ 已有 |

**建议：** 行业版作为 **「行业增强包」**（QB + 行业模块）挂载在通用引擎上；或 **统一入口** 先选模式与角色，再选是否加载行业模块。

## 7. 系统架构（技术草案）

### 7.1 前端

- **问卷壳**：React/Vue 或继续单页 H5；题目 **100% 由配置驱动**。  
- **断点续答**：`localStorage` + 服务端草稿（登录态企业用户）。  

### 7.2 后端

- **POST /submissions**：写入答案 JSON，返回维度分与等级。  
- **GET /questionnaires?role=&sale_mode=**：拉取题目包。  
- **GET /report/:submissionId**：渲染结构化报告（可先模板引擎，后接 LLM 仅润色）。  

### 7.3 内容管理

- **Excel → JSON 导入脚本**（openpyxl 一次性迁移 + 版本 diff）；后续以 **Admin 界面** 或 Git 管理 YAML/JSON 为主。

### 7.4 与纷享 / 门户

- 门户卡片：**通用诊断** 与 **行业诊断** 分链或统一落地页；`utm` / `employee_id` 回传 CRM（与现有 `crm_workflow.md` 对齐）。
